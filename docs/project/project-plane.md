# Guia dos Achados — Project Documentation

# Overview

Guia dos Achados is an AI-assisted affiliate discovery platform focused on generating highly optimized product recommendation pages for search engines and AI-based search systems.

The platform combines:
- SEO-focused content generation
- marketplace integrations
- AI-assisted ranking and recommendation
- analytics-driven discovery
- high-performance web architecture

The main goal of the platform is to create scalable, high-quality recommendation pages that rank well on Google and are also relevant for AI-powered search experiences.

The project will initially focus on categories such as:
- Construction
- Cars and Motorcycles
- Technology

The platform is designed to evolve into a scalable intelligent recommendation ecosystem.

# SEO Strategy

SEO is one of the core foundations of the platform.

The project should prioritize:
- fast loading pages
- static generation whenever possible
- structured metadata
- semantic HTML
- schema.org support
- optimized internal linking
- long-tail keyword targeting

The platform should focus heavily on:
- "Top 10" pages
- comparison pages
- buying guides
- FAQ-based pages
- problem-solving searches

Examples:
- Top 10 Marble Saws
- Best Bluetooth Headphones
- Bosch vs Makita Drill Comparison
- How to Choose a Good Motorcycle Helmet

---

# AI Search Optimization

The platform should also optimize content for AI-powered search systems.

The content architecture should:
- answer direct user questions
- contain structured summaries
- provide concise recommendations
- use semantic organization
- include contextual product data

The objective is to increase the chances of:
- AI assistants referencing the platform
- AI-generated search results using the content
- recommendation snippets including platform links

# Performance Strategy

Performance is one of the highest priorities of the platform.

The system should prioritize:
- static generation
- incremental static regeneration
- optimized image delivery
- server-side metadata
- minimal JavaScript
- lazy loading
- cache optimization

The project should avoid:
- unnecessary client-side rendering
- heavy frontend bundles
- excessive hydration
- unoptimized images
- slow API dependencies
